SCVMM における WORKGROUP クラスターのサポート状況をまとめた記事になります。
<br>

<!-- more -->
## はじめに
皆様こんにちは、System Center サポートチームの 保科 です。 
Windows Server 2025 では、過去のバージョンでは不可能だった WORKGROUP サーバー同士でのクラスターを構築した際にライブ マイグレーションを利用することができるようになりました。
Windows Server 2025 Hyper-V は SCVMM 2025 で新しく管理対象として登録することができるようになり、ご要件上クラスターはドメイン参加できないという環境も多いかと思います。
本記事では、この WORKGROUP クラスターというものが SCVMM で管理することができるか？という観点をお伝えします。

## 結論
大変恐縮ですが、SCVMM 2025 では現状ライブ マイグレーションをサポートする WORKGROUP クラスターの管理を行うことができません。
2026 年 6 月時点で最新版の UR1 を適用している場合でも同様で、現時点では SCVMM ではライブ マイグレーションをサポートする WORKGROUP クラスターの管理はサポート外の構成となります。
公開情報ではこちらが明記されておりませんでしたので、ブログでご紹介としました次第です。

## 原因
単体の WORKGROUP Hyper-V サーバーでは、ライブ マイグレーションは利用することができません。
また、Windows Server 2022 までの WORKGROUP クラスターにおいては、クラスター構成後もライブ マイグレーションを利用することができませんでした。
これが Windows Server 2025 の WORKGROUP クラスターから、ライブ マイグレーションがサポートされるようになりました。
詳しくはこちらの公開情報で解説されています。
[ワークグループ クラスターを使用してライブ マイグレーションを行う](https://learn.microsoft.com/ja-jp/windows-server/virtualization/hyper-v/manage/live-migration-workgroup-cluster?tabs=powershell)

ライブ マイグレーションをご利用の際においては認証方式の指定が必須です。
ドメイン環境では、CredSSP もしくは Kerberos 認証のどちらかを使用することになります。
WORKGROUP クラスターでは、この認証方式は PKU2UFallbackNTLM となります。

SCVMM では、クラスターのライブ マイグレーション認証方式を文字列ではなく値として、各ホストから取得します。
PKU2UFallbackNTLM は Windows Server 2025 の WORKGROUP クラスター用に新しく追加された認証方式で、その内部上の値は 4 となります。
ただし、SCVMM 側では現状この値は 0 (CredSSP) もしくは 1 (Kerberos) しか扱うことができません。
現状の SCVMM の実装上扱うことができないライブ マイグレーションの認証方式設定値が WORKGROUP クラスター内の各ホストから取得されてしまい、その結果例外が発生する状況となります。

## 発生する現象
例外の発生によって以下の現象が発生することを確認しています。
- WORKGROUP クラスターの登録自体が失敗する。
SCVMM の管理対象としての登録自体は行われている状況に見られますが、実際には例外の発生によってその登録は失敗しており、そのクラスターの管理も行うことができません。
- 定期的に SCVMM サーバー上の vmmservice.exe プロセスがクラッシュし、そのたびに SCVMM サーバーの機能が停止する。

## 回避方法
現時点ではライブ マイグレーションをサポートする WORKGROUP クラスターは絶対に SCVMM の管理対象に追加しない、以外の回避方法はございません。
一度ライブ マイグレーションをサポートする WORKGROUP クラスターを SCVMM の管理対象に登録すると、SCVMM サーバーの機能が定期的に停止する現象が発生してしまいます。
ご不便をお掛けしてしまい申し訳ございませんが、クラスターを管理する場合は以下の運用をいただけると幸いです。
- Windows Server 2025 のクラスターはドメイン参加してから SCVMM の管理対象に登録する。
- WORKGROUP クラスターの利用とそれを SCVMM で管理することが必須である場合は Windows Server 2022 以前のバージョンで WORKGROUP クラスターを構成する。
こちらの構成の場合、ライブ マイグレーションを利用することができなくなる点にご注意が必要となります。
- ライブ マイグレーションをサポートする WORKGROUP クラスターのご利用が必須な場合は SCVMM にそのクラスターを管理対象として登録しない。
